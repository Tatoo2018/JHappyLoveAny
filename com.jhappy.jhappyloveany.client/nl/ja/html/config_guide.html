JHappyLoveAny 設定ガイド

このガイドでは、プロジェクトでインテリジェントなコード補完を有効にするための jhappyqueries.xml の設定方法について説明します。

ファイルの配置場所: jhappyqueries.xml は、Eclipse プロジェクトのルートディレクトリに配置してください。

1. XML 構造

設定ファイルは、バリデーション用の DTD 定義を含むシンプルな XML 形式を使用します。

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE queries [
   <!ELEMENT queries (#PCDATA|query)*>
   <!ATTLIST queries completionMatch (prefix | contains) "contains">
   <!ELEMENT query (#PCDATA)>
   <!ATTLIST query filepath CDATA #IMPLIED trim (yes | no) "yes" type (xml | properties) "xml" description CDATA #IMPLIED>
]>
<queries completionMatch="contains">
    <!-- ここにクエリを記述します -->
</queries>

2. クエリ属性の詳細

属性名 説明
filepath 対象ファイルを指定するための正規表現（例: .*\.xml$）。
type ファイル形式: xml または properties。
trim 抽出されたキーから前後の空白を削除するかどうか（デフォルト: yes）。
description プロジェクト・プロパティ画面のテーブルに表示される短い説明文。

3. 設定例

XML キーの補完（XPath を使用）

XML の属性（例: <item id="my_key">）からキーを抽出するには、CDATA セクション内に XPath 式を記述します。

<query filepath=".*\.xml$" type="xml"><![CDATA[//@id]]></query>

Properties ファイルの補完

標準的な .properties ファイルの場合、XPath は不要です。ファイルパターンの指定のみで動作します。

<query filepath="src/.*\.properties$" type="properties" />
